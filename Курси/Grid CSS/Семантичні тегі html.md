Ось список основних семантичних тегів, розбитих за ролями. Вони допомагають структурувати документ так, щоб він був зрозумілим і для браузерів, і для розробників.

### Структурні теги (Верхній рівень)

- **`<header>`** — "шапка": вступна частина сторінки або розділу (логотип, навігація).
    
- **`<nav>`** — блок із навігаційними посиланнями.
    
- **`<main>`** — унікальний головний контент сторінки (може бути лише один на сторінку).
    
- **`<footer>`** — "підвал": інформація про автора, копірайт, контакти.
    

### Теги контенту

- **`<section>`** — тематичний розділ документа (наприклад, "Про нас", "Послуги").
    
- **`<article>`** — самостійний, незалежний блок тексту (пост у блозі, новина, картка товару).
    
- **`<aside>`** — бічна панель або додатковий контент, що опосередковано стосується теми (реклама, цитати).
    

### Теги для деталей

- **`<figure>`** — групує медіа-контент (зображення, графіки) з підписом.
    
- **`<figcaption>`** — заголовок або опис для вмісту всередині `<figure>`.
    
- **`<time>`** — позначає дату або час у машиночитаному форматі.
    
- **`<mark>`** — виділяє частину тексту (як маркером).
    
- **`<details>`** — створює розкривний список (віджет).
    
- **`<summary>`** — видимий заголовок для тегу `<details>`.
    

Використання цих тегів замість звичайних `<div>` значно покращує SEO та доступність (Accessibility) вашого сайту.
